🔷 Arrays & Hashing

Start Learning
What is it?

Definition: Arrays are contiguous blocks of memory storing elements of the same type. Elements are accessed via 0-based indices, allowing O(1) constant-time random access no matter the array size. This is their biggest strength compared to linked lists.

Why It Matters: Arrays are fundamental. 60% of interview problems involve arrays. Most problems taught first because they're simple yet powerful. Understanding arrays deeply gives you a foundation for every other data structure.

Key Operations & Complexity: Access element: O(1) | Search (unsorted): O(n) | Insert/Delete: O(n) because you need to shift elements | Sort: O(n log n) using merge/quick sort.

Core Patterns to Master: (1) Two Pointers - Use left & right pointers moving inward. Solves: sorted pair problems, palindrome checks, container-with-most-water. (2) Sliding Window - Maintain window of elements, slide right boundary to expand, left to contract. Solves: longest substring, max elements in k-size window. (3) Prefix Sum - Precompute cumulative sums for O(1) range queries. Solves: subarray sum, 2D matrix sum. (4) Hash Map - Store frequency/index, eliminate nested loops. (5) Kadane's Algorithm - Find max subarray sum in O(n).

Common Mistakes to Avoid: Off-by-one errors in loops, forgetting array bounds, not handling edge cases (empty array, single element), modifying array while iterating, assuming 1-based indexing.

Interview Tips: Always clarify: Can I modify input array? Is array sorted? What about duplicates? Start with brute force O(n²) nested loops, then optimize to O(n) or O(n log n). Test: empty array, single element, all same elements, negatives, large numbers.

🔤 Strings

Start Learning
What is it?

Definition: Strings are immutable sequences of characters. In languages like Java/Python, once created, you cannot modify individual characters—you must create a new string. This immutability affects algorithm design (use StringBuilder for repeated concatenation in Java).

Why It Matters: Strings appear in ~20% of interviews. Similar to arrays but with character-specific tricks. Teaches you frequency counting, pattern matching, and two-pointer techniques.

Key Operations & Complexity: Access char at index: O(1) | substring: O(k) for k-length substring | concatenation: O(n+m) | string operations generally O(n) where n = string length.

Core Patterns to Master: (1) Character Frequency Counting - Use hash map to track char counts. Solves: anagram, valid rearrangement. (2) Sliding Window - Find longest/shortest substring with condition. Solves: longest substring without repeating chars. (3) Two Pointers - Validate palindromes, reverse strings. (4) Pattern Matching - KMP algorithm for efficient substring search. (5) Hash vs Sort - Sort for anagrams (a^2=a^3 after sort), hash for frequencies.

Interview Tips: Clarify: Are inputs lowercase only, or include uppercase/digits/special chars? Case-sensitive comparison? Palindrome includes only alphanumeric (skip spaces/punctuation)? Use frequency array [26] for lowercase a-z, or hash map for general. Common: valid anagram, longest substring without repeating, longest palindromic substring, valid palindrome with cleanup.

Performance Pitfall: String concatenation in loops using "+=" is O(n²) in Java. Always use StringBuilder append() for O(n) total time.

🔍 Binary Search

Start Learning
What is it?

Definition: Binary Search is a divide-and-conquer algorithm that halves the search space each iteration. Works ONLY on sorted data. By comparing the middle element, you eliminate half the remaining elements, achieving O(log n) complexity—massive improvement over O(n) linear search.

Why It Matters: Reduces time from O(n) to O(log n). For 1 million elements, O(n) = 1M steps, O(log n) = 20 steps. Interview favorite because it teaches patience (don't jump to first solution) and precision (handle all edge cases).

Master Template: left = 0, right = n-1 | while(left <= right) { mid=left + (right-left)/2; if(arr[mid]==target) return mid; if(arr[mid]> target) right = mid-1; else left = mid+1 } | return -1. Avoid: mid = (left+right)/2 can overflow. Always use mid = left + (right-left)/2.

Variations to Know: (1) Standard - Find exact element. (2) Left Boundary - Find first occurrence. (3) Right Boundary - Find last occurrence. (4) Rotated Sorted Array - Determine which side is sorted, eliminate that side. (5) Find Peak - Mountain array, go toward higher side. (6) Answer-Space Search - Instead of searching element location, search answer range (e.g., min capacity).

Common Pitfalls: Not checking if array is sorted first. Infinite loops (mid calculation bugs). Wrong boundary updates. Off-by-one errors when finding boundaries. Forgetting target not found case (-1).

Interview Strategy: Always verify array is sorted. Draw diagram for rotated/peak problems. Code template confidently. Test: empty array, single element, target at boundary, target not found.

⛓️ Linked Lists

Start Learning
What is it?

Definition: Linked List is a dynamic data structure where each node contains data and a reference (pointer) to the next node. Unlike arrays, NO random access—you must traverse from head. But insertion/deletion at known position is O(1) (just update pointers), while array requires O(n) shifting.

Why It Matters: Tests pointer manipulation and linked thinking. Essential foundation for graphs and advanced data structures. Teaches you to handle null references carefully.

Key Operations & Complexity: Access element at index: O(n) traversal | Insert/Delete at known position: O(1) after reaching it | Search: O(n) | Space: O(n).

Core Patterns to Master: (1) Two Pointers (Slow/Fast) - Slow moves 1 step, fast moves 2. Detects cycles (Floyd's), finds middle (slow is middle when fast reaches end). (2) Reversal - Three pointers: prev, current, next. Reverse direction of pointers. (3) Dummy Node - Fake head node avoids special-casing removal of actual head. (4) Merge - Combine sorted lists, pick smaller node from each. (5) Cycle Detection - Slow/fast meet = cycle exists.

Interview Tips: ALWAYS draw the diagram. Pointer updates are error-prone. Verbalize: 'I'm updating current.next to point to previous'. Handle: null checks, single node, empty list, cycles. Practice: reversal, cycle detection, merge two lists, remove nth node, intersection point.

Key Insight: Linked lists shine for insert/delete at known position. Poor for random access. Most interviews: use dummy node and two-pointer technique.

📦 Stacks

Start Learning
What is it?

Definition: Stack is a LIFO (Last In First Out) abstract data structure. Imagine a stack of plates—you add and remove from the top. The last element added is the first removed. Core operations: push (add to top) O(1), pop (remove from top) O(1), peek (view top) O(1).

Why It Matters: Powers DFS (depth-first search). Essential for expression evaluation, undo functionality, function call stack. Interview problems test if you recognize 'go back' or 'undo' patterns.

Real-World Uses: (1) Call stack in programming languages—function calls push frame, return pops. (2) Browser back button—each visited page pushed, back button pops. (3) Undo/Redo in editors—undo pops from undo stack, redo pushes to redo stack.

Core Patterns to Master: (1) Validation - Matching parentheses/brackets. (2) Monotonic Stack - Maintain increasing/decreasing stack. Solves: next greater element, largest rectangle. (3) Expression Evaluation - Convert infix to postfix, evaluate postfix. (4) DFS Traversal - Use stack instead of recursion to avoid stack overflow. (5) Min Stack - Track minimum while supporting push/pop/getMin all O(1).

Implementation Detail: Use array/ArrayList (with dynamic resizing) or LinkedList. ArrayList push/pop at end is O(1) amortized.

Interview Tip: See 'previous', 'matching', 'validate', 'undo' keywords → think Stack!

🚀 Queues

Start Learning
What is it?

Definition: Queue is a FIFO (First In First Out) abstract data structure. Imagine a line at a store—first person in is first served. Core operations: enqueue (add to back/rear) O(1), dequeue (remove from front/head) O(1), peek (view front) O(1).

Why It Matters: Powers BFS (breadth-first search) for level-order traversal of trees/graphs. Models real-world queuing (job scheduling, print queue). Interview problems test level-order/distance-based thinking.

Real-World Uses: (1) Printer jobs—queue incoming print requests, process in order. (2) Customer service—customers join queue, served in order. (3) BFS in graphs—start node enqueued, expand neighbors level by level. (4) Task scheduling—processes queued, executed based on priority/order.

Core Patterns to Master: (1) BFS Traversal - Level-order tree/graph traversal. (2) Distance Tracking - Find shortest path in unweighted graph. (3) LRU Cache - Combine queue + hash map for O(1) operations. (4) Deque (Double-Ended Queue) - Add/remove from both ends. Solves: sliding window maximum.

Implementation Detail: Use circular array or LinkedList. Circular array efficiently reuses space. Java: LinkedList or ArrayDeque (preferred for deque operations).

Interview Tip: See 'level-order', 'distance', 'shortest path', 'neighbors' → think Queue + BFS!

🌳 Trees

Start Learning
What is it?

Definition: Tree is a hierarchical, acyclic data structure with a root node and child nodes. Each node (except root) has one parent. Key terms: root (top), parent (node with children), child, leaf (no children), subtree, height (longest path to leaf), depth (distance from root). Binary Tree: max 2 children per node.

Why It Matters: Trees model hierarchical data (file systems, DOM). ~20% of interviews. Binary Search Trees enable fast search. Teaches recursion and divide-and-conquer thinking.

Traversal Methods (Critical to Master): (1) Inorder (Left-Root-Right) - BST inorder gives sorted sequence. (2) Preorder (Root-Left-Right) - Root first, good for copying tree, serialization. (3) Postorder (Left-Right-Root) - Children before parent, good for deletion, space calculation. (4) Level-Order (BFS) - Visit nodes level by level, use queue.

Tree Types: (1) Binary Search Tree (BST) - Left subtree < node < right subtree. Enables O(log n) search if balanced. (2) Balanced Trees - AVL, Red-Black ensure height O(log n). (3) Complete Binary Tree - All levels filled except possibly last level, filled left to right. (4) Full Binary Tree - Every node has 0 or 2 children.

Core Patterns to Master: (1) Recursion - Solve base case (leaf), combine child results. (2) DFS - Traverse depth-first using recursion or stack. (3) BFS - Level-order using queue. (4) Path Problems - Path sum, lowest common ancestor. (5) Serialization - Convert tree to string/array and back.

Complexity: Balanced binary tree: search O(log n), insert/delete O(log n). Skewed (like linked list): all O(n). Always check if tree is balanced.

Interview Strategy: Draw tree clearly. Recursion template: define return type, base case, recursive case. Test: empty tree, single node, skewed tree, balanced tree.

🔗 Graphs

Start Learning
What is it?

Definition: Graph is a non-linear data structure with vertices (nodes) and edges (connections). Directed: edges have direction (A→B) vs Undirected: bidirectional. Weighted: edges have costs vs Unweighted: uniform cost. Can have cycles unlike trees. Models networks, social connections, maps.

Why It Matters: Graph problems are challenging but appear in interviews. Tests algorithmic thinking. Solves: social networks, maps, routing, recommendations.

Representations: (1) Adjacency List - Map/Array of lists. Sparse graphs: O(V+E) space, good for most problems. (2) Adjacency Matrix - 2D array. Dense graphs: O(V²) space. Avoid unless problem specifies dense.

Core Algorithms to Master: (1) DFS (Depth-First Search) - Visit deep before broad, use stack/recursion. O(V+E). (2) BFS (Breadth-First Search) - Visit level by level, use queue. O(V+E). Finds shortest path in unweighted graphs. (3) Dijkstra's Algorithm - Shortest path in weighted graphs (non-negative weights). O((V+E) log V) with heap. (4) Topological Sort - Order vertices so edges point forward. Only for directed acyclic graphs (DAGs). (5) Cycle Detection - DFS color marking (white/gray/black) or Union-Find. (6) Union-Find - Efficiently find connected components, detect cycles, used in Kruskal's MST.

Interview Approach: (1) Identify: Is it asking for path? Connected components? Shortest path? Cycle? (2) Choose algorithm: Unweighted shortest→BFS. Weighted→Dijkstra. Rank order→Topological sort. Components→Union-Find. (3) Implement carefully: Build adjacency list first, handle both directed and undirected.

Common Interview Problems: Clone graph (DFS/BFS copy), course schedule (topological sort), number of islands (union-find or DFS), network delay (Dijkstra), critical connections (bridge-finding algorithm).

⚡ Dynamic Programming

Start Learning
What is it?

Definition: Dynamic Programming solves problems by breaking into overlapping subproblems, solving each once, storing results to avoid recomputation. Combines optimal substructure (optimal solution uses optimal subsolutions) with overlapping subproblems (same subproblems repeat). Two approaches: (1) Memoization (Top-Down) - Recursion + cache. (2) Tabulation (Bottom-Up) - Iterative, build from base cases up.

Why It Matters: Hardest DSA topic. Appears in ~15% of interviews. Tests if you can recognize problems beyond surface. Transforms exponential O(2ⁿ) to polynomial O(n²) or O(n³).

When To Use DP: (1) Problem asks for optimal value (max/min). (2) Problem has overlapping subproblems. (3) Problem has optimal substructure. (4) Constraints allow polynomial time/space.

5-Step DP Approach: (1) Define State - What does dp[i] represent? E.g., dp[i] = max profit up to day i. (2) Base Case - Initial values. E.g., dp[0] = prices[0]. (3) Recurrence Relation - How does dp[i] depend on previous states? E.g., dp[i] = max(dp[i-1], price[i] - min_price). (4) Compute - Iterate or recurse. (5) Return - Usually last dp value or maximum seen.

Classic DP Problems (Must Know): (1) Fibonacci - Simplest, f(n) = f(n-1) + f(n-2). (2) Coin Change - Min coins for amount. (3) Knapsack (0/1) - Max value with weight constraint. (4) Longest Increasing Subsequence (LIS) - dp[i] = max length ending at i. (5) House Robber - Max loot without robbing adjacent houses. (6) Edit Distance - Min operations to transform one string to another.

Memoization vs Tabulation: Memoization easier to code (recursive), debug, but risk stack overflow. Tabulation more efficient (iterative), guaranteed not to revisit, easier space optimization. Practice both.

Complexity Trade-off: Memoization O(n²) time, O(n) space. If you recognize pattern, optimize space: rolling array (only track last 2 rows) reduces O(n²) space to O(n).

Interview Strategy: (1) Start with brute force (exponential). (2) Identify overlapping subproblems (evidence DP helps). (3) Draw state table for small example. (4) Define recurrence carefully. (5) Code iterative version, test transitions. (6) Optimize space if possible. Most don't optimize; solve problem first.

🏆 Heaps

Start Learning
What is it?

Definition: Heap is a complete binary tree satisfying heap property: (1) Max-Heap - Parent >= children. Root is maximum. (2) Min-Heap - Parent <= children. Root is minimum. Complete means all levels filled except possibly last, filled left-to-right. Stored as array where parent at index i has children at 2i+1 (left), 2i+2 (right).

Why It Matters: Powers Priority Queues. Efficient for repeatedly finding max/min. Heapsort is O(n log n) in-place sort. Interview problems (top-k, median, merge k lists) often solved with heaps.

Core Operations & Complexity: Insert: O(log n) - add at end, bubble up. Delete root: O(log n) - swap last with root, bubble down. Find root: O(1) - immediate access. Heapify (convert array): O(n).

Key Algorithms: (1) Insert - Add element at end, bubble up (compare with parent, swap if needed). (2) Delete/Extract-Min - Remove root, move last element to root, bubble down (compare with smaller child, swap down). (3) Heapify - Convert unsorted array to heap in O(n) (better than n inserts = O(n log n)).

Core Patterns to Master: (1) Top-K Problem - Keep min-heap of size k, O(n log k). (2) Kth Largest/Smallest - Min-heap for largest (remove k-1 times), max-heap for smallest. (3) Merge K Sorted Lists - Min-heap of heads, pop min, push next from that list. (4) Median of Data Stream - Max-heap for smaller half, min-heap for larger half, balance sizes. (5) Heap Sort - Heapify array O(n), extract n times O(n log n), in-place sort.

Interview Tips: Use language's PriorityQueue (min-heap default in Java). For max-heap, negate values or use custom comparator. Clarify: find top-k largest or smallest? Duplicates allowed? Negative numbers? Practice: kth largest, median stream, merge k lists, meeting rooms (heap variant).

Why Better Than Sorting? Sorting k largest from n elements: sort all O(n log n). Heap approach: n-element operation O(n log k) where k << n. Spatial wins matter in interviews (explain trade-off).
