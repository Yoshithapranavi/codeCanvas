🔷 Arrays & Hashing

▶ Start Learning

💡 What's This About?

Arrays are contiguous blocks of memory. Store elements side-by-side, enabling instant O(1) access via index. 60% of interview problems involve arrays! Master arrays = master interviews.

⚡ Why Learn This?

  • Foundation skill: Every algorithm relies on understanding arrays
  • Interview staple: Easy → Medium → Hard progression on arrays
  • Optimal access: O(1) random access beats all others
  • Gateway to patterns: Two Pointers, Sliding Window, Prefix Sum all use arrays

⏱️ Time Complexity Cheat Sheet

Access element: O(1) ⚡ Instant

Search (unsorted): O(n) - Must scan all

Insert/Delete: O(n) - Shift elements needed

Sort: O(n log n) - Use merge/quick sort

🎯 5 Core Patterns Every Developer Needs

1️⃣ Two Pointers

Move left & right inward. Solves: sorted pairs, palindromes, container-with-most-water.

2️⃣ Sliding Window

Expand right, contract left. Solves: longest substring, max in k-size window.

3️⃣ Prefix Sum

Precompute cumulative sums. Solves: range sum, 2D matrix sum queries in O(1).

4️⃣ Hash Map

Store frequency/index. Eliminates nested loops, converts O(n²) to O(n).

5️⃣ Kadane's Algo

Find max subarray sum in O(n). Elegant dynamic programming approach.

⚠️ Gotchas to Avoid

  • Off-by-one errors in loops (i < n vs i <=n)
  • Forgetting array bounds (IndexOutOfBoundsException)
  • Not handling edge cases: empty array, single element, duplicates
  • String concatenation in loop = O(n²), use StringBuilder
  • Modifying array while iterating (ConcurrentModificationException)
🎤 Interview Strategy: Always ask: "Can I modify input? Is it sorted? Duplicates allowed?" Start with brute force O(n²), then optimize. Test: empty, single element, all same, negatives, very large numbers.

🔤 Strings

▶ Start Learning

💡 What's This About?

Strings are immutable sequences of characters. Once created, you can't change them—must create new ones. This immutability completely changes how you solve problems compared to arrays!

⚡ Why Learn This?

  • 20% of interviews: String manipulation questions appear frequently
  • Character-specific tricks: Frequency counting, pattern matching unique to strings
  • Real-world problems: Validation, parsing, text processing
  • Performance pitfall: String concatenation in loops = O(n²) disaster!

⏱️ Time Complexity Cheat Sheet

Access char at index: O(1) ⚡ Instant

Substring creation: O(k) - k = length of substring

Concatenation ("+="): O(n²) in loops - USE StringBuilder!

String comparison: O(n) - Must check each char

🎯 4 Critical Patterns

1️⃣ Frequency Counting

Use hash map or array [26]. Solves: anagrams, valid rearrangement, character frequency.

2️⃣ Sliding Window

Find longest/shortest substring with property. Solves: longest substring without repeats.

3️⃣ Two Pointers

Validate palindromes, reverse strings efficiently.

4️⃣ Sort or Hash

For anagrams: sort both then compare, or use frequency arrays.

⚠️ Common Mistakes

  • Concatenation loop: String s = ""; for(...) s += char; = O(n²), use StringBuilder
  • Not handling case sensitivity: clarify if comparison is case-sensitive
  • Forgetting special chars in palindrome: "A man, a plan, a canal: Panama" = palindrome (ignoring punctuation)
  • Assuming lowercase only: might have uppercase, digits, special characters
🎤 Interview Strategy: Clarify: Lowercase only or mixed case? Only alphanumeric or special chars too? Using frequency array [26] for a-z? Always use StringBuilder for multiple concatenations. Test: empty, single char, all same, palindrome edge cases.

🔍 Binary Search

▶ Start Learning

💡 What's This About?

Divide and conquer for sorted data. Half the search space each step. O(log n) vs O(n) = 20 steps vs 1M steps for 1 million elements. This is optimization magic!

⚡ Why Learn This?

  • Interview favorite: Tests precision and edge case handling
  • Performance game-changer: O(n) → O(log n) for sorted arrays
  • Gateway pattern: Foundation for advanced algorithms
  • Real-world use: Searching massive databases, time-series data

6 Variations to Master:

1. Standard (find exact) | 2. Left boundary (first occurrence) | 3. Right boundary (last occurrence)

4. Rotated array | 5. Mountain/peak | 6. Answer-space search (capacity, speed, etc)

🎯 Master Template (Copy This!)

left = 0, right = n-1
while (left <= right) {
    mid = left + (right - left) / 2
    if (arr[mid] == target) return mid
    if (arr[mid] > target) right = mid - 1
    else left = mid + 1
}
return -1  // not found

⚠️ CRITICAL: Use mid = left + (right-left)/2, NOT (left+right)/2 (overflow bug!)

🎤 Interview Strategy: Always verify array is sorted first! Draw diagram for rotated/peak problems. Code template confidently. Test: empty, single element, target at boundary, target not found, duplicates.

⛓️ Linked Lists

▶ Start Learning

💡 What's This About?

Dynamic data structure with node chains. Each node = data + pointer to next. NO random access (O(n) to reach any node), BUT O(1) insert/delete at known position. Pointer manipulation is tricky—draw diagrams!

⚡ Why Learn This?

  • Tests pointer thinking: Abstract pointer manipulation skills
  • Foundation for graphs: Graph nodes also have pointers
  • No random access: Different algorithm thinking than arrays
  • Interview challenge: Errors are common—nulls, cycles, reversals

⏱️ Time Complexity Cheat Sheet

Access element: O(n) 🐌 Must traverse

Insert at position: O(1) if position known, else O(n)

Delete at position: O(1) if position known, else O(n)

Search: O(n) - Linear scan

🎯 5 Essential Patterns

1️⃣ Fast/Slow Pointers

Slow = 1 step, Fast = 2 steps. Detects cycles, finds middle node.

2️⃣ Reversal

Three pointers: prev, curr, next. Reverse pointer direction smoothly.

3️⃣ Dummy Node

Fake head avoids special-casing removal of actual head node.

4️⃣ Merge Sorted

Combine two sorted lists, pick smaller from each, advance.

5️⃣ Cycle Detection

Floyd's algorithm: if fast meets slow, cycle exists!

🎤 Interview Strategy: DRAW THE DIAGRAM ALWAYS. Verbalize pointer updates: "I'm making current.next point to previous." Test: empty list, single node, cycle, null handling. Practice reversal until automatic.

📦 Stacks & Queues

▶ Start Learning

💡 What's This About?

Stack = LIFO (Last In First Out). Like a stack of plates—remove from top. Last plate added = first removed. Queue = FIFO (First In First Out). Like a line—first person in = first served.

⚡ Why Learn Stacks?

  • DFS power: Stacks are behind all depth-first searches
  • Expression evaluation: Solve math expressions, reverse Polish notation
  • Undo/Redo: Browser back button, editor undo functionality
  • Pattern spotter: See "matching", "previous", "undo" → think Stack!

⚡ Why Learn Queues?

  • BFS power: Queues are behind all breadth-first searches
  • Level-order traversal: Visit graph/tree level by level
  • Shortest path: Find quickest route in unweighted graphs
  • Pattern spotter: See "level-order", "distance", "neighbors" → think Queue!

🎯 Stack Patterns (5 Must-Know)

1️⃣ Matching Brackets

Valid parentheses: push '(', pop when see ')'. Empty = valid.

2️⃣ Monotonic Stack

Keep increasing/decreasing. Solves: next greater element, largest rectangle.

3️⃣ Min Stack

Track minimum while push/pop all O(1). Use two stacks: data + min.

4️⃣ Expression Eval

Infix to postfix conversion, then evaluate postfix.

5️⃣ DFS Without Recursion

Use stack instead of recursion to avoid stack overflow.

🎯 Queue Patterns (3 Must-Know)

1️⃣ BFS Traversal

Level-order tree/graph traversal. Use queue to process level by level.

2️⃣ Shortest Path

Find shortest path in unweighted graph. Distance = levels from start.

3️⃣ Deque (Double-Ended)

Add/remove from both ends. Solves: sliding window maximum with monotonic deque.

🎤 Interview Strategy: See "previous element", "next greater", "matching" → Stack. See "level order", "distance", "neighbors" → Queue. Test: empty, single element, edge cases where structure is fully filled/empty.

🌳 Trees

▶ Start Learning

💡 What's This About?

Hierarchical data structure. Root at top, children below. Each node (except root) has one parent. Binary Tree: Max 2 children per node. Binary Search Tree: left < node < right (enables fast search).

⚡ Why Learn This?

  • ~20% of interviews: Tree problems are common
  • Real-world everywhere: File systems, DOM, databases use trees
  • Recursion mastery: Tree problems teach elegant recursive thinking
  • Traversal algorithms: 4 different ways to visit nodes

🎯 4 Traversal Methods (Master All!)

1️⃣ Inorder (L-Root-R)

BST inorder = SORTED sequence. Most commonly used for BST.

2️⃣ Preorder (Root-L-R)

Root first. Good for copying tree, serialization, cloning.

3️⃣ Postorder (L-R-Root)

Children first. Good for deletion, calculating tree properties.

4️⃣ Level-Order (BFS)

Level by level. Use queue. Good for vertical/right view, zigzag.

⏱️ Complexity Guide

Balanced tree: Search O(log n), Insert O(log n), Delete O(log n)

Skewed tree (like linked list): All operations O(n) - WORST CASE!

Always check: Is this tree balanced? If skewed, complexity degrades!

🎯 5 Essential Patterns

1️⃣ Recursion Template

Base case (leaf), solve for left, solve for right, combine results.

2️⃣ Path Problems

Path sum, max path sum, ancestor questions.

3️⃣ Lowest Common Ancestor

Find deepest common ancestor of two nodes. Recursive or iterative.

4️⃣ Serialization

Convert tree to string and back. Test cloning, comparison.

5️⃣ Validate BST

Check if tree satisfies BST property (tricky with range validation).

🎤 Interview Strategy: DRAW THE TREE CLEARLY. Define return type for recursion. What should each recursive call return? How do you combine left/right results? Test: empty tree, single node, skewed tree, balanced tree.

🔗 Graphs

▶ Start Learning

💡 What's This About?

Most general structure: Vertices + edges (can have cycles, multiple connections). Directed: edges have direction. Weighted: edges have costs. Models networks, social connections, maps.

⚡ Why Learn This?

  • Hardest DSA topic: Tests algorithmic sophistication
  • Real-world power: GPS routing, social networks, recommendations
  • Complex algorithms: DFS, BFS, Dijkstra, topological sort, union-find
  • Interview challenge: Can you recognize graph problem in disguise?

🎯 6 Core Algorithms (Master These!)

1️⃣ DFS

Depth-first. Use stack/recursion. O(V+E). Good for: connected components, cycles.

2️⃣ BFS

Breadth-first. Use queue. O(V+E). Good for: shortest path (unweighted), distance.

3️⃣ Dijkstra

Shortest path (weighted, non-negative). Heap-based. O((V+E) log V).

4️⃣ Topological Sort

Order vertices so edges point forward. Only for DAGs. Interview favorite!

5️⃣ Union-Find

Efficiently track connected components, detect cycles. Nearly O(1) per operation.

6️⃣ Cycle Detection

DFS color marking (white/gray/black) or union-find for undirected.

🎯 Interview Decision Tree

  • Shortest path in unweighted? → BFS
  • Shortest path in weighted? → Dijkstra (if no negatives)
  • Need rank/order? → Topological sort
  • Connected components? → Union-Find or DFS
  • Detect cycle? → DFS or Union-Find
🎤 Interview Strategy: Always build adjacency list first (even for matrix input). Handle both directed and undirected. Name your graph variables clearly (adj, neighbors, graph). Test: disconnected components, self-loops, duplicate edges.

⚡ Dynamic Programming

▶ Start Learning

💡 What's This About?

Solve overlapping subproblems once, cache results. Transforms exponential O(2^n) → polynomial O(n²) or O(n³). The hardest DSA topic, but also the most powerful!

⚡ Why Learn This?

  • Interview differentiator: ~15% of interviews, separates strong from average
  • Algorithmic power: Takes impossible O(2^n) and makes it solvable
  • Real-world: Resource optimization, scheduling, financial planning
  • Hardest topic: Expect 9-12 months to master

🔥 Why DP Matters:

Brute force: Try all choices, returns exponential O(2^n). For n=20: 1M checks. For n=40: 1 trillion checks!

DP: Calculate once, reuse. O(n²) for n=40: 1600 operations. 625 MILLION times faster! 🚀

🎯 5-Step DP Approach (Always Follow This!)

Step 1: Define State

What does dp[i] represent? E.g., dp[i] = max profit up to day i, dp[i][j] = ways to reach (i,j)

Step 2: Base Case

Initial values. E.g., dp[0] = prices[0], dp[0][0] = 1 way

Step 3: Recurrence Relation

How does dp[i] depend on previous states? E.g., dp[i] = min(dp[i-1]+1, dp[i-2]+2)

Step 4: Fill Table

Iterate from base case up, fill dp table bottom-up (tabulation) or top-down (memoization)

Step 5: Extract Answer

Usually dp[n-1] or max/min seen during computation

🎯 6 Classic Problems (MUST KNOW!)

  • Fibonacci: f(n)=f(n-1)+f(n-2). Simplest DP. Memoization vs Tabulation comparison.
  • Coin Change: Min coins for amount. State: dp[i] = min coins for amount i.
  • 0/1 Knapsack: Max value with weight limit. 2D DP: items × capacity.
  • Longest Increasing Subsequence: dp[i] = max length ending at i. Compare all j < i.
  • House Robber: Max loot without adjacent houses. dp[i] = max(dp[i-1], dp[i-2]+current).
  • Edit Distance: Min operations to transform string A→B. Classic 2D DP problem.
🎤 Interview Strategy: (1) Start with brute force, identify overlapping subproblems. (2) Draw state table for small example (n=5). (3) Define state clearly on paper before coding. (4) Code recurrence carefully—most errors here. (5) Optimize space if time permits. Most don't optimize; solve problem first.

🏆 Heaps / Priority Queue

▶ Start Learning

💡 What's This About?

Complete binary tree with heap property. Max-heap: parent ≥ children. Min-heap: parent ≤ children. Find max/min instantly O(1), but extraction O(log n). Perfect for top-k problems!

⚡ Why Learn This?

  • Powers priority queues: Process items by priority, not order received
  • Top-k problems: Find k largest/smallest efficiently
  • Real-world: Task scheduling, event streams, marketplace (prioritize urgent)
  • Interview patterns: Kth largest, median stream, merge k lists

⏱️ Time Complexity Cheat Sheet

Insert element: O(log n) - Add at end, bubble up

Extract max/min: O(log n) - Remove root, move last element, bubble down

Find max/min: O(1) ⚡ Instant - It's the root!

Heapify (convert array): O(n) - Better than n inserts = O(n log n)

🎯 5 Essential Patterns

1️⃣ Top-K Problem

Keep min-heap of size k. For k largest: O(n log k). Much faster than sorting!

2️⃣ Kth Largest

Min-heap for largest (extract k-1 times), max-heap for smallest.

3️⃣ Merge K Sorted

Min-heap of heads, pop min, push next from that list. O(n log k).

4️⃣ Median Stream

Max-heap (smaller half) + min-heap (larger half). Balance sizes.

5️⃣ Heap Sort

Heapify O(n), extract n times O(n log n). In-place sort alternative.

🎯 Why Heap Beats Sorting

Find top-10 from 1 million items:

Sorting approach: Sort 1M items O(n log n) = 20M operations, then take first 10. WASTEFUL!

Heap approach: Keep min-heap of 10, scan 1M items O(n log k) = 200K operations. 100x faster!

🎤 Interview Strategy: Java: Use PriorityQueue (min-heap by default). For max-heap, negate or use custom comparator. Clarify: top-k largest or smallest? Duplicates allowed? Test: heap with 1 element, all same values, negative numbers.
